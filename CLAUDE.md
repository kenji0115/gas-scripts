このファイルは Claude Code がこのリポジトリで作業する際のガイドラインです。

## プロジェクト概要

Google Apps Script (GAS) のスクリプトを管理するリポジトリです。
スプレッドシート・ドキュメント・Gmail・カレンダーなどの Google サービスを
自動化する `.gs` / `.js` スクリプトを格納します。

## ディレクトリ構成

- 各スクリプトはプロジェクトごとにサブディレクトリへ配置する
- GAS プロジェクトのメタ情報は `appsscript.json` に記述する
- ローカルと GAS を同期する場合は [clasp](https://github.com/google/clasp) を利用する

## コーディング規約

- 関数名・変数名は意図が伝わる命名にする（`doGet`, `onOpen` などの予約名を除く）
- 外部 API キーやトークンは **コードに直接書かず**、
  `PropertiesService`（スクリプトプロパティ）から取得する
- 1 ファイルが肥大化したら役割ごとにファイルを分割する
- 既存スクリプトのスタイル（インデント・コメント量・命名）に合わせる

## Git 運用ルール

**コードを変更するたびに、必ず GitHub へプッシュすること。**

具体的な手順:

1. 変更が一区切りついたら、その都度コミットする
2. コミット後はすぐに `git push` で GitHub のリモートへ反映する
3. 変更を溜め込まず、こまめにコミット・プッシュする
4. コミットメッセージは変更内容が分かる日本語または英語で簡潔に書く

```bash
git add -A
git commit -m "変更内容の説明"
git push
```

### 補足

- まだ Git リポジトリが初期化されていない場合は `git init` から始める
- リモートが未設定の場合は `git remote add origin <GitHubのURL>` を設定する
- 機密情報（API キー・トークン・個人情報）はコミットしない。
  必要に応じて `.gitignore` で除外する

## clasp を使う場合の補足

```bash
clasp pull   # GAS 側の最新を取得
clasp push   # ローカルの変更を GAS へ反映
```

clasp で GAS に反映したあとも、上記の Git 運用ルールに従って
GitHub へのプッシュを忘れずに行うこと。
